Features
User can search the recipe by entering its name in the text box.
API response contains recipes of all the items matching with that word. I am returning 4 recipe's images and titles returned from it.

Then on click on the 'Recipe' button I am displaying recipe details same as in the cuisine tab
In case of a random/wrong input I am displaying appropriate error message.

On click on the 'Recipe' button I am making an entry in a database to keep the counter of viewed recipe. For the Recipe being viewed for the first time I am making an entry with its RecipeID and counter equalt to 1. For already viewed recipe I am incrementing its counter by 1.
Learning
I have learnt to use user's input value to make an API request.
I have learnt to indicate user's on their wrong input in search box.
I have handled the cases of no result found and api limit reached differenly so the user will know for what particular reason the output is not being generated.

I have learnt to make a web service to make entry into database on user action on html pages.
I have used stored procedure to make entry in database table.
